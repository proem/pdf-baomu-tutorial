# 保姆教程的写作心法

这份 skill 面向“想自己折腾但不懂命令行”的用户。写作风格融合了两套原则：

- **真、通、暖**：用比喻降维，用“我们/大家”拉近距离
- **Claude Code 时代**：把所有难点都转化成“发给 AI 的中文指令”

## 壹 开篇怎么写

开篇第一段是灵魂。模式：**场景或设问 → 本文要解决的问题 → 建立共鸣**

**好的开篇**（来自 OpenList 教程“零 这份教程是给谁看的”）：

> 挂载 115 这件事本来应该很难，但因为你有 Claude Code，它就变得不太难了。
> 
> 如果你正在看这份教程，大概率属于下面这类人：
> - 你有一个 115 网盘，里面存了很多电影、剧集或者资料
> - 你想把这些东西接到 Emby、Jellyfin、Infuse 这样的“家庭影院”软件里看

**不好的开篇**：

> 本文将介绍如何使用 OpenList 挂载 115 网盘。OpenList 是一款开源的网盘聚合工具…

（说了相当于没说，读者已经知道了，否则不会来看。）

## 贰 章节节奏

每一章严格遵循这五步：

1. **概念** — 这一章在讲什么（一两句话）
2. **为什么重要** — 不懂怎样，懂了又怎样
3. **怎么理解** — 用日常比喻或类比降维
4. **具体怎么做** — Claude Code 引导块 + 步骤序列
5. **校验** — 做完怎么知道对了

**第三步（比喻降维）是和普通教程区别最大的地方**，一定要给。

示例：

> **概念**：要让 OpenList 访问你的 115，你要先去“115 开放平台”办一张通行证。
> 
> **为什么重要**：Cookie 那一套已经完全不能用了，走开放平台是现在唯一稳定的路。
> 
> **怎么理解**：这个通行证叫 Token，就像是 115 官方给你的“工牌”，刷这张牌进门不会被
> 当成小偷。
> 
> **具体怎么做**：[紫色 Claude Code 引导块]
> 
> **校验**：授权成功后，页面会显示你的 Access Token 和 Refresh Token，两串长字符。

## 叁 比喻库（可复用）

| 技术概念 | 日常比喻 |
|---------|---------|
| Token | 身份证 / 工牌 / 会员卡 |
| Cookie | 临时出入证 |
| Docker 容器 | 打包好的快递盒 |
| 风控 | 保安盯着你别搞事 |
| 302 重定向 | “那个东西不在我这，在隔壁” |
| STRM 文件 | 一张“视频在那边”的便利贴 |
| API 请求 | 按门铃 |
| 反向代理 | 前台小哥，帮你传话 |
| 配置文件 | 遥控器的设置 |
| 端口 | 房间号 |
| 挂载 | 把远方的硬盘当成家里的 U 盘 |

碰到没在表里的概念，**现编一个生活比喻**，但检查两件事：
- 比喻是不是大多数人都懂的生活场景？
- 比喻映射得准不准，不会误导？

## 肆 Claude Code 引导块的写法

这是这份风格的**灵魂元素**。每个动手点都应该有一个。

**好的引导块**：

```
标题：如果你不知道怎么登录，直接问它：

指令：我的 NAS 的 IP 是 192.168.x.x，用户名是 xxx，我想 SSH 登录上去。
     帮我一步一步做，不要一次发太多命令。
```

**不好的引导块**：

```
标题：执行以下命令登录

指令：ssh -p 22 user@192.168.x.x
```

区别在哪？
- **好的**是“跟 Claude Code 说的人话”，带着用户的上下文、期望、约束
- **不好的**是“直接让用户敲命令”——这就不是小白教程了

### 长指令的写法

超过一句话的指令，用编号列表：

```
请帮我部署 OpenList。要求如下：

1. 在 /opt/openlist 目录下创建一个 compose.yml
2. 使用镜像 openlistteam/openlist:latest
3. 端口映射 5244:5244
4. 部署完成后告诉我初始密码和访问地址
```

**写作要点**：
- 开头一句话描述整体目标
- 要求 3-6 条为宜，太多会吓到 Claude Code 和用户
- 最后一条常常是“告诉我 XXX”——让 AI 主动反馈结果
- 尽量带上**现实约束**（路径、版本号、环境等）

## 伍 警告框的尺度感

三种警告强度，不要乱用：

### 💡 蓝色提示（callout-tip）

用于：给信心、解释原理、小贴士。

```html
<div class="callout callout-tip">
  <strong>💡 一个重要的心态</strong>
  遇到看不懂的报错，别慌，直接发给 Claude Code。
</div>
```

### ⚠️ 黄色警告（callout，无后缀类）

用于：需要注意但不致命，做错了能恢复。

```html
<div class="callout">
  <strong>⚠️ 注意别选错</strong>
  列表里还有一个叫"115 Cloud"的，那是老的 Cookie 方式，不要选。
</div>
```

### 🚨 红色严重警告（callout-warn）

用于：做错了有严重后果（数据丢失、账号封禁、Token 泄露）。

```html
<div class="callout callout-warn">
  <strong>⚠️ 立刻把这两个值复制下来！</strong>
  页面一关就找不回来了。
</div>
```

**判断标准**：
- 能通过“再点一次”恢复 → 黄色
- 恢复需要折腾或等待 → 红色
- 根本不影响结果，只是知识补充 → 蓝色

## 陆 附录怎么写

附录是读者反复回来看的地方，要做成**速查手册**而不是“知识堆积”。

**核心表格**：`我想做什么 → 发给 Claude Code 的话`

```
| 我想做什么 | 发给 Claude Code 的话 |
| ---------- | ------------------- |
| 重启 OpenList | "重启 openlist 容器" |
| 看最近日志 | "看 openlist 最近 100 行日志" |
| 更新版本 | "把 openlist 拉最新版重启" |
```

这张表比任何命令参考手册都有用——因为它保持了“小白不用懂命令”的一致性。
